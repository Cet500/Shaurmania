# Админ-панель приложения `main`

## Обзор

Все модели приложения `main` зарегистрированы в админ-панели Django с настройками для удобного управления.

## Структура админки

```
main/admin/
├── __init__.py        # Регистрация всех админов
├── admin.py           # Базовые настройки
├── user.py            # Админ для пользователей
├── shaurma.py         # Админ для товаров
├── review.py          # Админ для отзывов
├── location.py        # Админ для локаций
├── achievement.py     # Админ для достижений
├── news.py            # Админ для новостей
└── stock.py           # Админ для акций
```

## Зарегистрированные модели

### Пользователи

- **`User`** — управление пользователями
- **`UserAvatar`** — управление аватарами
- **`UserAddress`** — управление адресами
- **`UserSocialLink`** — управление социальными ссылками

### Товары

- **`Shaurma`** — управление товарами
- **`ShaurmaCategory`** — управление категориями
- **`ShaurmaImage`** — управление изображениями товаров

### Контент

- **`Review`** — управление отзывами
- **`Location`** — управление локациями
- **`Achievement`** — управление достижениями
- **`UserAchievement`** — управление достижениями пользователей
- **`News`** — управление новостями
- **`NewsTag`** — управление тегами новостей
- **`Stock`** — управление акциями
- **`Delivery`** — управление доставкой

## Типичные настройки админки

### list_display

Отображение полей в списке объектов:

```python
list_display = ['name', 'price', 'is_available', 'created_at']
```

### list_filter

Фильтры в правой панели:

```python
list_filter = ['is_available', 'category', 'created_at']
```

### search_fields

Поля для поиска:

```python
search_fields = ['name', 'description']
```

### ordering

Сортировка по умолчанию:

```python
ordering = ['-created_at']
```

### list_per_page

Количество объектов на странице:

```python
list_per_page = 25
```

## Доступ к админ-панели

Админ-панель доступна по адресу:

```
http://localhost:8000/admin-panel/
```

Требуется:

- Аутентификация
- Права `is_staff=True` или `is_superuser=True`

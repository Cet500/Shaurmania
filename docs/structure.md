# Структура документации

Документация проекта организована по приложениям для удобства навигации и понимания архитектуры.

## Описание разделов

### Корневой уровень

- **`index.md`** — главная страница с обзором проекта и навигацией
- **`structure.md`** — описание структуры документации (этот файл)
- **`architecture.md`** — общая архитектура проекта, технологический стек
- **`installation.md`** — установка, настройка, управление данными
- **`quick-reference.md`** — краткий справочник команд и полезных ссылок

### `applications/` — Документация по приложениям

Каждое Django-приложение имеет свою папку с документацией:

#### `applications/main/`

Основное приложение проекта:

- **`index.md`** — обзор приложения, назначение, основные функции
- **`models.md`** — все модели приложения (User, Shaurma, Review, Location и т.д.)
- **`views.md`** — описание представлений и URL-маршрутов
- **`admin.md`** — настройки админ-панели

#### `applications/cart/`

Приложение корзины и заказов:

- **`index.md`** — обзор функциональности корзины
- **`models.md`** — модели Cart, Order, Promocode

#### `applications/geodata/`

Приложение геоданных:

- **`index.md`** — обзор системы геоданных
- **`models.md`** — модели географической иерархии и адресов
- **`services.md`** — сервисы геокодирования

#### `applications/security/`

Приложение безопасности:

- **`index.md`** — обзор системы безопасности
- **`models.md`** — модели SecurityDevice, SecurityAuthLog, SecurityNotice

#### `applications/api/`

API приложение:

- **`index.md`** — обзор API
- **`endpoints.md`** — документация всех эндпоинтов

### `development/` — Руководство для разработчиков

Расширенное руководство по разработке:

- **`index.md`** — обзор раздела, навигация
- **`getting-started.md`** — начало работы, настройка окружения
- **`code-style.md`** — стиль кода, конвенции, примеры
- **`models.md`** — работа с моделями, создание, валидация
- **`views.md`** — работа с представлениями, формы, обработка ошибок
- **`testing.md`** — тестирование, pytest, Factory Boy
- **`optimization.md`** — оптимизация запросов, кэширование
- **`debugging.md`** — отладка, логирование, инструменты

## Как использовать

1. **Для быстрого старта:** начните с `installation.md` и `quick-reference.md`
2. **Для понимания архитектуры:** изучите `architecture.md`
3. **Для работы с конкретным приложением:** перейдите в соответствующую папку в `applications/`
4. **Для разработки:** используйте раздел `development/`

## Принципы организации

- **Разделение по приложениям** — каждая документация приложения изолирована
- **Табличное представление моделей** — модели представлены в виде таблиц для наглядности
- **Практические примеры** — каждый раздел содержит примеры кода
- **Пошаговые инструкции** — сложные темы разбиты на шаги

## Обновление документации

При добавлении нового функционала:

1. Обновите документацию соответствующего приложения в `applications/`
2. Если добавляется новая модель — обновите `models.md` в папке приложения
3. Если меняется архитектура — обновите `architecture.md`
4. Если добавляются новые практики — обновите раздел `development/`
